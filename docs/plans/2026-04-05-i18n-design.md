# i18n Design — KickerCoach App

## Ziel

Internationalisierungs-Infrastruktur aufbauen mit `react-i18next`. Alle ~300-400 hardcoded German Strings in Translation-Dateien externalisieren. Vorerst nur deutsche Sprachdatei, aber jederzeit erweiterbar.

## Entscheidungen

| Frage | Entscheidung |
|-------|-------------|
| Library | react-i18next + i18next + i18next-browser-languagedetector |
| Sprachen (initial) | Nur Deutsch |
| Dateistruktur | Separate JSON pro Feature-Namespace |
| Domain-Konstanten | Labels wandern in i18n, Farben/Werte bleiben in constants.ts |
| Sprach-Umschalter | Ja, in Settings-Seite (zeigt vorerst nur Deutsch) |
| URL-basiertes Routing | Nein (kein /de/ Prefix) |

## Architektur

```
src/
  i18n/
    index.ts                  # i18next Init, LanguageDetector, Fallback 'de'
    locales/
      de/
        common.json           # Navigation, Buttons, Dialoge, Fehler, shared Labels
        home.json             # HomePage
        learn.json            # LearnMode, CardGrid, CardDetail, SearchFilter
        train.json            # TrainMode, DrillEditor, SessionBuilder, Timer, Journal
        players.json          # PlayerList, PlayerForm, GoalForm, Notes, Teams
        plan.json             # MatchPlanEditor, MatchPlanList
        board.json            # BoardMode, Toolbar, SceneManager
        analytics.json        # Charts, Filter, PlayerComparison
        settings.json         # SettingsPage, Sprach-Umschalter
```

## Namespace-Konventionen

- Keys in **camelCase**: `addPlayer`, `searchPlaceholder`, `emptyState.title`
- Nesting max 2 Ebenen: `{ "form": { "nameLabel": "Name" } }`
- Interpolation: `{{count}}`, `{{name}}`
- Domain-Labels via Prefix: `constants.difficulty.beginner`, `constants.phase.warmup`

## Komponenten-Integration

```tsx
// Vorher
<h1>Spieler</h1>
<button>+ Spieler hinzufuegen</button>

// Nachher
const { t } = useTranslation('players');
<h1>{t('title')}</h1>
<button>{t('addPlayer')}</button>
```

## Domain-Konstanten Migration

`src/domain/constants.ts` Label-Maps (DIFFICULTY_LABELS, PHASE_LABELS, MOOD_LABELS, etc.) werden durch i18n-Lookup-Funktionen ersetzt:

```tsx
// Vorher
DIFFICULTY_LABELS[drill.difficulty]

// Nachher
t(`constants.difficulty.${drill.difficulty}`)
```

Die Farb-Maps (CATEGORY_COLORS, MOOD_COLORS, SKILL_COLORS etc.) bleiben in constants.ts.

## Sprach-Umschalter

- `<LanguageSelector />` Komponente in Settings-Seite
- Dropdown mit verfuegbaren Sprachen (initial nur "Deutsch")
- Speicherung via i18next-browser-languagedetector (localStorage)
- Sprachwechsel laedt Namespaces nach, kein Page-Reload noetig

## Dependencies

```
i18next                         # Core
react-i18next                   # React-Bindings (useTranslation, Trans, I18nextProvider)
i18next-browser-languagedetector # Spracherkennung aus localStorage/Browser
```

## Scope-Grenzen

- Kein serverseitiges i18n (reine PWA)
- Kein URL-basiertes Routing (/de/, /en/)
- Zustand Store bleibt unveraendert (i18n ist View-Layer only)
- Keine Pluralisierung in Phase 1 (Deutsch hat einfache Plural-Regeln)
- Keine RTL-Unterstuetzung
